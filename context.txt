# Contesto progetto Bynari — Prompt per nuova istanza

Stai continuando lo sviluppo di **Bynari**, una webapp SaaS per la gestione di allenamenti in palestra. Di seguito trovi tutto il contesto necessario per continuare senza interruzioni.

---

## Stack tecnico

- **Framework:** Next.js 14 (App Router) + TypeScript
- **Styling:** Tailwind CSS v4 (utility classes + style inline con oklch)
- **Backend/DB:** Supabase (PostgreSQL + Auth + Storage)
- **Deploy:** Vercel — URL live: `my-gym-plan-delta.vercel.app`
- **Repository:** `github.com/Coe1998/mygymplan`
- **Icone:** FontAwesome (`@fortawesome/react-fontawesome`)
- **Grafici:** Recharts
- **Colori brand:**
  - Accent arancione: `oklch(0.70 0.19 46)` = `#f97316`
  - Background: `oklch(0.13 0 0)`
  - Card: `oklch(0.18 0 0)`

---

## Architettura ruoli

Il sistema ha 3 ruoli distinti con aree separate:

| Ruolo | Area | Route |
|---|---|---|
| `coach` | Gestisce clienti, schede, esercizi, analytics | `/coach/*` |
| `cliente` | Riceve schede dal coach, si allena, vede progressi | `/cliente/*` |
| `atleta` | Utente autonomo (free/pro), crea proprie schede | `/atleta/*` |

---

## Schema database Supabase (tabelle principali)

```sql
profiles (id, full_name, role: coach|cliente|atleta, piano: free|pro, 
          coach_code TEXT UNIQUE, coach_status: pending|approved)

coach_clienti (coach_id, cliente_id, created_at)
coach_inviti (id, coach_id, cliente_id, stato: pending|approvato|rifiutato)

esercizi (id, coach_id, nome, muscoli TEXT[], descrizione)
schede (id, coach_id, nome, descrizione, is_template)
scheda_giorni (id, scheda_id, nome, ordine)
scheda_esercizi (id, giorno_id, esercizio_id, serie, ripetizioni, recupero_secondi, note, ordine)

assegnazioni (id, coach_id, cliente_id, scheda_id, data_inizio, data_fine, 
              attiva BOOL, pdf_alimentare_url TEXT)

sessioni (id, cliente_id, giorno_id, assegnazione_id, data, completata, durata_secondi)
log_serie (id, sessione_id, scheda_esercizio_id, peso_kg, ripetizioni, completata)

misurazioni (id, cliente_id, data, peso_kg, note)
checkin (id, cliente_id, data, energia, sonno, stress, motivazione, note)
messaggi (id, coach_id, cliente_id, testo, mittente_ruolo, created_at)
```

## Funzionalità implementate (complete)

### Area Coach
- Dashboard con stats (clienti, schede, assegnazioni)
- Gestione clienti: aggiungi via email + link invito univoco + approvazione manuale
- Schede: CRUD + drag&drop riordino esercizi + upload PDF alimentare per assegnazione
- **Filtro gruppi muscolari** nella modale aggiungi esercizio (chip cliccabili + ricerca testuale)
- Analytics: overview, benessere, peso, attività — query parallele con `Promise.all` (4 query totali invece di 4×N)
- Impostazioni account (modifica nome, password, cancellazione)
- Bottom nav mobile con bottone "Altro" che apre mini-menu (Impostazioni + Esci)
- Sistema inviti: `coach_code` unico → link `/join/[code]` → richiesta pending → approvazione

### Area Cliente
- Dashboard con scheda attiva + ultimi allenamenti
- Allenamento: timer recupero, log serie, note esercizi (modal bottom sheet)
- Progressi: grafici peso max/volume per esercizio, misurazioni, foto, check-in
- **Riepilogo settimanale** con navigazione corrente/precedente + modal storico completo
- Share card allenamento e settimanale (canvas PNG) con logo Bynari reale + font Big Shoulders Display
- Tutorial collassabile "Come condividerlo su Instagram" nelle share card

### Area Atleta
- Piano free/pro con paywall modal
- Limiti free: 1 scheda, 3 giorni per scheda, no progressi, no checkin
- Schede ed esercizi propri (libreria personale)
- **Filtro gruppi muscolari** nella modale aggiungi esercizio (identico al coach)

### Sistema generale
- PWA completa: manifest, service worker, icone, banner iOS
- `loading.tsx` su tutte le route principali (spinner arancione)
- Auth con redirect per ruolo, coach pending page
- Logo PNG reale in sidebar, login, register, landing, share card
- Icone PWA con mark centrato (compensazione offset visivo)

## Dettagli tecnici importanti

### Pattern query Supabase
```typescript
// GIUSTO — parallelo
const [res1, res2, res3] = await Promise.all([
  supabase.from('...').select('...'),
  supabase.from('...').select('...'),
  supabase.from('...').select('...'),
])

// DA EVITARE — sequenziale in loop
for (const cliente of clienti) {
  const { data } = await supabase.from('...').select('...')  // 4×N query!
}
```

### Next.js 15 — params asincroni
```typescript
// GIUSTO — params è una Promise in Next.js 15
export default async function Page({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
}
```

### Supabase — evitare errori su query vuote
```typescript
// GIUSTO — non lancia errore se non trova risultati
.maybeSingle()

// DA EVITARE — lancia errore se non trova risultati
.single()
```

### Share card (canvas)
- Font: **Big Shoulders Display** (Google Fonts, caricato dinamicamente)
- Logo: `<img id="__bynari_logo__">` nascosta, `drawImage` sul canvas
- Dark: `Bynari_W1.png` (h=38px), Light: `Bynari_B1.png` (h=62px, più grande per compensare padding)
- Scale canvas: ×3 per alta risoluzione

### Piano atleta — `src/lib/piani.ts`
```typescript
export const LIMITI_FREE = {
  max_schede: 1,
  max_giorni_per_scheda: 3,
  progressi: false,
  checkin: false,
}
```

## Bug risolti in questa sessione (non ripetere)

1. **Link invito "non valido"** — doppia causa:
   - `params` non awaited (Next.js 15): `const { code } = await params`
   - `.single()` → `.maybeSingle()` sulle query della join page
   - Case mismatch: il codice è salvato MAIUSCOLO nel DB ma l'URL può essere minuscolo → `code.toUpperCase()` prima della query

2. **Icona PWA non centrata** — `MonoB_WO1.png` aveva 48px vuoti sopra e 22px sotto → compensazione offset visivo calcolata dinamicamente

3. **Bottom nav coach overflow** — 6 voci + logout non stavano in una riga → sostituito logout con bottone "Altro ···" che apre mini-menu popup con Impostazioni + Esci

4. **Analytics lenti** — loop sequenziale 4×N query → `Promise.all` con 4 query aggregate

5. **`'use client'` mancante** in CoachSidebar dopo aggiunta `useState`

## Note di stile / convenzioni

- Tutti i colori tramite `oklch()` inline, mai classi Tailwind per i colori brand
- Componenti server dove possibile, `'use client'` solo dove necessario (hooks, interattività)
- Le sidebar desktop usano `hidden lg:flex`, la bottom nav usa `lg:hidden`
- I file `loading.tsx` sono identici in tutte le route (spinner arancione animato)
- Niente `<form>` HTML negli artifact React — usare `onClick` handlers
- Apostrofi nelle stringhe JSX vanno escaped: `sull\'immagine` oppure usare `&apos;`
- Virgolette dentro stringhe TSX: usare `&quot;` oppure template literal
